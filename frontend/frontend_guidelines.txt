Core coaching flow (the backend drives this, but the frontend must support the UI for it):

After logging a brew, the user rates it 1-10. The rating determines the coaching mode:

Diagnosis (1-4): "What's wrong?" → User picks from 7 taste symptoms (sour, bitter, weak/watery, flat/no sweetness, muddy/silty, woody/papery, hollow) → Backend returns a method-specific fix
Improvement (5-6): User can either pick a symptom OR select improvement goals (e.g., "more sweetness", "more body") → Backend returns a moderate fix
Refinement (7-8): User selects up to 2 improvement goals → Backend returns a nudge-level tweak
Lock-in (9-10): Backend congratulates and offers to save the recipe as a favourite
The backend may also return freshness caveats (appended after the technique fix, never instead of it) and trend observations (improving, worsening, oscillating, plateau) based on brew history.
